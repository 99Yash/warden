- [x] M16 — Init/review alignment per ADR-0032 (shipped on the `m16` branch, 2026-05-17, with post-dogfood fixes). Lines up the two main agents — `warden review` (M14/M15) and `warden init` (M6) — across document updates. Production-RAG framing the user reads (Arpit Bhayani's *Production RAG: Document Management & Indexing Strategies*) names the failure modes M16 fixes: chunk identity, content-hash-gated re-embed, delete semantics, doc registry. Pre-M16 failure modes: (a) edited files leak their old chunks forever (the old `chunk_hash`es from the previous `file_sha` never get pruned); (b) deleted files leak their *entire* chunk + embedding + merkle footprint (`merkleStore.diff().removed[]` is computed but unread); (c) review surfaces stale state via the M6 banner but refuses to refresh — user has to manually re-run `warden init`; (d) `chunks.file_path` is first-writer-wins (`INSERT OR IGNORE` on the content PK), not authoritative — pruning by `file_path` would corrupt files that share content. Five deliverables: (a) new `file_chunks (file_path, chunk_hash, file_sha, indexed_at)` junction table with PK `(file_path, chunk_hash)` + indexes on both PK columns — the authoritative file→chunks registry; (b) `reconcileFiles({files, removed, maxUsdBudget?, ...})` primitive at `packages/core/src/init/reconcile.ts` — embeds outside the SQLite transaction so HTTP failure leaves the file at its pre-call state; per-file atomic DB writes; batched orphan prune at the call boundary (`DELETE FROM chunks WHERE chunk_hash NOT IN (SELECT chunk_hash FROM file_chunks)` + embeddings cascade); (c) implicit refresh on every `warden review` — `det-priors.ts` calls `reconcileFiles()` with the stale subset (from `merkleStore.diff()`'s `changed ∪ added`) plus `removed[]`; env knob `WARDEN_REVIEW_REFRESH_MAX_USD` has consumer fallback `0.25`; over-budget files are skipped and surfaced as one actionable degraded entry pointing at `warden init` for a full refresh; (d) one-shot auto-backfill from `chunks.file_path` when `file_chunks` is empty + `chunks` is non-empty (gated by `index_meta.file_chunks_backfilled_at`); approximate-but-bounded — first subsequent reconcile overwrites; (e) `semantic.ts` attribution rerouted through `FileChunksStore.getFilesForHashes()` join so a chunk shared across two files emits one max-aggregated `SemanticHit` per file. `runInit()` thins to a wrapper over `reconcileFiles({files: all_walked, removed: merkle.diff().removed})`. `chunks.file_path` + `chunks.file_sha` stay vestigial — kept *only* for the auto-backfill source-of-truth window; future cleanup ADR may drop them. Zero `CategoryEnum` / `PRIORITY_ORDER` / `SourceType` / public `CommentSet` impact. No new flag, no new verb — opt-out via `WARDEN_REVIEW_REFRESH_MAX_USD=0`.
